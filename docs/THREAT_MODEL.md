# 威胁模型（MVP）

> 范围：Web 端 + API + OTS 盖章 + Gmail SMTP 验证 + MetaMask 签名。原文与密钥只在用户本地。

## 1. 安全目标
- **完整性**：承诺与时间戳不可被篡改。
- **可验证性**：第三方能独立核验承诺与时间戳。
- **隐私**：原文与 salt/nonce 不离开用户设备。
- **可用性**：盖章与验证在合理时间内可完成。

## 2. 资产清单
- 原文内容（高敏）
- salt/nonce（高敏）
- 钱包签名（中敏）
- 承诺 hash（低敏）
- OTS 证明文件（低敏）
- 邮箱验证记录（中敏）
- SMTP 账号凭据（高敏）
- 服务端签名密钥（高敏）

## 3. 信任边界
- **用户设备**：可信存放原文与 salt/nonce（但可能被本机恶意软件破坏）
- **API/Worker**：不可信原文，仅处理承诺与证明
- **第三方 OTS 日历**：不可信，仅作为时间戳来源

## 4. 主要威胁与缓解

### 4.1 伪造承诺/签名
- 威胁：攻击者伪造签名或重复提交旧签名。
- 缓解：
  - 签名内容包含 `commitment + client_nonce + timestamp`
  - 服务端验证签名与时间戳范围
  - 服务器拒绝重复 nonce

### 4.2 证据包篡改
- 威胁：证据包被修改导致验证误判。
- 缓解：
  - 证据包 `manifest.json` 加服务端签名
  - 验证页校验签名与 OTS 证明

### 4.3 邮箱验证被劫持
- 威胁：攻击者拿到验证链接。
- 缓解：
  - magic link 单次有效，短有效期（如 15 分钟）
  - 同一邮箱绑定频率限制

### 4.4 OTS 日历不可用
- 威胁：盖章失败或延迟。
- 缓解：
  - 使用多日历提交
  - 失败重试队列

### 4.5 用户丢失 salt/nonce
- 威胁：无法揭示/验证。
- 缓解：
  - 提示导出本地备份文件
  - 明示风险（不可恢复）

### 4.6 XSS/前端注入
- 威胁：泄露 salt/nonce 或篡改承诺。
- 缓解：
  - CSP（禁止内联脚本）
  - 严格依赖锁版本
  - 输入输出统一编码

### 4.7 API 滥用与刷接口
- 威胁：资源耗尽、恶意提交。
- 缓解：
  - IP 限流
  - 基本验证码（可选）
  - 审计日志

### 4.8 SMTP 凭据泄露
- 威胁：邮件账户被滥用。
- 缓解：
  - 只使用应用专用密码
  - 仅本地 `.env` 保存，不入库
  - 定期轮换

## 5. 残留风险（可接受）
- 本地恶意软件可篡改原文/窃取 salt/nonce（用户设备层面无法完全防护）。
- 时间戳仅证明“存在时间”，不能证明作者身份。

## 6. 监控与审计
- 记录所有承诺提交、邮箱验证、盖章请求的审计日志。
- 对失败盖章、异常提交设置告警。

## 7. 安全假设
- 用户设备能妥善保存原文与 salt/nonce。
- 钱包私钥由钱包管理，不被服务端接触。
- SMTP 账号与应用密码妥善保管。
